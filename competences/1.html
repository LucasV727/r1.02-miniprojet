Compétence 1 : Réaliser un développement d’application

Apprentissages critiques – Niveau 1

Développer des applications informatiques simples.

  • AC 1 : Implémenter des conceptions simples.
  • AC 2 : Élaborer des conceptions simples.
  • AC 3 : Faire des essais et évaluer leurs résultats en regard des spécifications.
  • AC 4 : Développer des interfaces utilisateurs.

Situations d’apprentissage et d’évaluation

SAÉ S1.01 : Implémentation d’un besoin client

Problématique : Créer tout ou partie d’une application simple.

Description : À partir d’un besoin exprimé par un client, l’objectif est de réaliser une application ainsi que les tests associés.

Cursus : 19h (12h projet tutoré + 7h formation complémentaire).

SAÉ S2.01 : Développement d’une application

Problématique : Concevoir une application avec interface graphique répondant à un contexte précis.

Description : Formaliser les besoins, proposer une conception, implémenter et tester le développement. Livrables : diagrammes, code, jeux d’essais.

Cursus : 29h (16h projet tutoré + 13h formation complémentaire).

R1.01

Initiation au développement

AC liés : AC 1, AC 2, AC 3

Algorithmes fondamentaux, manipulation de structures de données et premières notions de qualité (nommage, assertions, documentation…).

R1.02

Développement d’interfaces web

AC liés : AC 3, AC 4

Création de documents web répondant à des besoins client : HTML, CSS, maquettage et accessibilité (W3C/WAI).

R2.01

Développement orienté objets

AC liés : AC 1, AC 2, AC 3

Paradigme objet, conception détaillée (diagrammes de classes, séquences) et algorithmes sur structures de données.

R2.02

Développement d’applications avec IHM

AC liés : AC 2, AC 4

Programmation événementielle, conception d’interfaces, séparation vue/modèle et notions d’ergonomie.

R2.03

Qualité de développement

AC liés : AC 2, AC 3

Gestion des cas d’erreurs, tests unitaires, débogage et premiers outils de gestion de versions.
